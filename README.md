# ToDo Interview App

Консольное приложение для управления списком дел, реализованное на Python.  
Проект выполнен в рамках тестового задания.

## Возможности

- Добавление задач
- Удаление задач
- Отметка задач как выполненных
- Изменение задач (переименование, изменение статуса)
- Просмотр списка задач
- Сохранение данных между запусками в JSON
- Обработка ошибок и понятные сообщения пользователю

## Структура проекта

```
Todo_interview/
├── app.py # Точка входа
├── cli.py # Консольный интерфейс (UI)
├── service.py # Бизнес-логика
├── storage.py # Работа с JSON-хранилищем
├── models.py # Модель Task
├── requirements.txt # Зависимости (только стандартная библиотека)
├── .gitignore
└── tests/
    └── test_service.py
```
### Причины такого разделения

Разделение на модули было сделано осознанно, чтобы:

- разделить ответственность между слоями (UI / логика / хранение данных)

- упростить поддержку и чтение кода

- облегчить тестирование (бизнес-логика тестируется отдельно от CLI)

- упростить расширение функциональности

- приблизить структуру проекта к реальным production-решениям

Упрощённая версия (todo.py) добавлена в репозиторий намеренно — как иллюстрация эволюции решения от минимального прототипа к более масштабируемой архитектуре.

## Требования

- Python **3.9+**
- Внешние зависимости не требуются (используется стандартная библиотека)

## Установка и запуск

1. Клонировать репозиторий:

```
bash
git clone https://github.com/<your-username>/todo_interview.git
cd todo_interview
```

2. (Опционально) создать и активировать виртуальное окружение:

```
python -m venv .venv
.venv\Scripts\activate   # Windows
```

3. Запустить приложение:

```
python app.py
```

## Использование

После запуска отобразится интерактивное меню.

Все операции (удаление, выполнение, изменение) выполняются по номеру задачи в текущем списке,
внутренние id пользователю не показываются.

Данные автоматически сохраняются в файл tasks.json (он исключён из Git).

Приложение загружает состояние из JSON при запуске и дальше работает с данными в памяти.
Изменения сохраняются обратно в файл, но внешние изменения файла во время работы не отслеживаются.

## Тесты

Для бизнес-логики реализованы unit-тесты.

Запуск тестов из корня проекта:

```
python -m unittest -v
```

В репозитории также присутствует файл `todo.py`, содержащий упрощённую реализацию приложения **в одном файле**.
